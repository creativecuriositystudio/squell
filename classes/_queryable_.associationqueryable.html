as

  • as<T>(aliased: string | Queryable<T>): Queryable<T>

  • Inherited from Queryable.as

    • Defined in queryable.ts:208

    Alias a queryable under a different name.

    If you provide a string it will use that as a name, otherwise it will use the underlying name of a provided queryable. The more type-safe option is to alias to another queryable object, as that will let you store a function queryable under an actual queryable of the model. The queryable should be set to virtual if it's not meant to be synced to the database.

    Type parameters
